## Why this exists

The 2026-05-09 wave-10 closeout Chrome QA walkthrough exercised the appendix-#10 integrated flow end-to-end (rename a domain → rotate a binding's credentials → run heartbeat now → view audit log → view cost dashboard → change Lint cadence to bi-weekly → forget a source with impact preview) and surfaced wiring + UX gaps that the per-R-PR before/after pairs had missed:

- The R7 forget endpoint returned 503 `composition_incomplete` against the design-partner deployment ("Nie udało się załadować wpływu" on dialog open). R7 had wired the route's expectation of injected `deleteCap` + `forgetJobEnqueuer`, but `cli/src/provision/production-composition.ts` did not supply them — every R7 unit test passed because the test fixtures injected fakes; production composition simply never wired the real instances. **PRD §5 criterion 9 was structurally green but not actually green** until this was fixed.
- The `/Cost` dashboard recorded every OpenRouter (kimi) call as `$0.00` because `packages/shared/src/cost-tracker/pricing.ts` was missing entries for 13 catalog models — including `moonshotai/kimi-k2.6`, the model the design-partner deployment pins for all three tiers. Direct-vendor calls (Anthropic, OpenAI, Google) recorded correct prices; OpenRouter routes silently fell to `FALLBACK_PRICING`. The cost tracker emitted `cost-tracker.unknown_model` warnings, but the user-visible regression was the dashboard under-reporting, not the warning stream.
- The Activity-feed SSE reconnect loop kept thrashing on 401s after a stale PAT (a soft DoS path against the engine's own admin-auth surface). Tracked since appendix #9 as task #47.
- The partner-facing `pilot-runbook.md` had no coverage of wave-10 day-2 operations (R1–R7) and no documentation of the post-`git pull` upgrade procedures that wave-10's new migration (`0011_domains_disabled_at`) and new UI assets required.
- Two wave-10 modals (`SourceBindingDetail` edit mode ~700px tall and `DomainDetail` edit mode) pushed the bottom action row (Cancel / Save / Disable / Delete) below the fold at the QA viewport (1235×702). Operators couldn't see Save without resizing the window — broke the edit-flow muscle memory established earlier in the QA pass.

Wave-11 is the **pilot-cutover hardening wave**. Every PR closes one of the five gaps. None add new product surface. Three are essentially infrastructure or polish (W3 reconnect terminal state, W4 docs, W5 modal shell); two flip pre-tag blockers from red to green (W1 forget production wiring, W2 cost-tracker pricing). Scoping was driven through the planning agent + plan file at `~/.claude/plans/` rather than an on-disk `docs/plan-appendix/phase-a-11-*.md` scoping doc — the gap inventory was small enough and the per-PR scope clear enough from the QA findings that the closeout doc (this file) is the only on-disk record.

## Process notes

- **Five implementer subagents in parallel-friendly worktrees.** The five PRs were dispatched as five independent implementer agents working in separate worktrees (`.claude/worktrees/w-w1`, `w-w2`, `w-w3`, `w-w4`, `w-w5`), each with its own scoping prompt derived from the QA findings. W2, W3, W4, W5 were fully parallel-safe (zero file overlap with each other); W1 had an implicit dependency on the existence of R7 (`packages/shared/src/forget/planner.ts`, `cli/src/provision/production-composition.ts`) but no overlap with any other W-PR. Merge order was driven purely by which subagent finished its CI loop first, not dependency order.
- **CI cancellation pattern recurred for W1 four times before the toolchain timeout was bumped.** The new PGlite-backed composition tests in `packages/cli/tests/production-composition-r7-wiring.test.ts` push the per-package test run time past the original 25-min CI ceiling. The first three CI attempts on W1 cancelled at the 25-minute mark with no useful diagnostic; the fourth succeeded after `.github/workflows/` was updated to lift the timeout to 35 min. **Meta-finding worth recording**: PGlite-based composition tests have a non-trivial cold-start cost; future appendices that introduce similar boot-the-real-composition tests should plan for the timeout bump as part of the same PR rather than chasing four cancellations to discover it. (Other W-PRs were unaffected — W2 / W3 / W4 / W5 finished CI well within the original budget.)
- **No locale-merge-conflict pattern this wave.** Unlike appendix #10 (where R3, R6, R7 each rebased into `packages/ui/src/locales/{en,pl}.json` collisions), wave-11 had only W3 touching locales (`activity.feed.*` keys) and W5 not touching them at all; nothing to merge. The pattern from appendix #10 only fires when ≥2 PRs in a single wave land i18n strings, and wave-11's surface area was narrow enough to avoid it.
- **Closeout doc consolidation.** Per appendix-#10's PR #90 closeout pattern, the per-W-PR scattered entries in `CHANGES-v0.1.md` (the W1 follow-up bullet inside Appendix #10 R7, plus the standalone W2 + W5 entries that PR-W2 / PR-W5 added during their own merges) were consolidated into a clean Appendix #11 (W1–W5) section by this closeout PR. The scattered per-PR entries were removed; all detail preserved in the consolidated section.
